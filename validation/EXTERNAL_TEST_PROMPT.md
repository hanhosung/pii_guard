# 외부 LLM(Codex 등)으로 PII-Guard 테스트 텍스트 생성하기

이 문서는 **다른 LLM에게 테스트 텍스트를 생성시키는 프롬프트**와, 그 결과를 **PII-Guard에 입력해 채점**하는 방법을 설명한다.

핵심 원칙: 검출률(재현율/정밀도)을 채점하려면 **각 텍스트에 "무엇을 심었는지"(ground truth)가 라벨로 함께** 와야 한다. 그래서 LLM에게 **JSON**으로 (텍스트 + 심은 값 목록 + 함정)을 함께 내놓게 한다.

---

## 1. LLM에 붙여넣을 프롬프트 (복사해서 사용)

> 아래 블록 전체를 Codex/다른 LLM에 붙여넣으면 된다. `{N}`, `{KIND}` 만 원하는 값으로 바꾼다.

```text
너는 한국어 PII 탐지기 'PII-Guard'를 평가할 합성 테스트 데이터를 만든다.
목표: 현실적인 텍스트를 만들되, 그 안에 심은 모든 민감정보를 카테고리와 함께 정확히 라벨링한다.

[생성 대상]
- {KIND} 항목 {N}개. (VOC = 고객센터 문의 ~2000자 / LOG = 서버 console 로그 ~3000자)

[심을 수 있는 카테고리] (탐지기가 아는 20종)
- 시크릿(차단): API_KEY(sk-..., ghp_...), AWS_SECRET(AKIA+16), GCP_KEY(AIza...),
  TOKEN(JWT eyJ...), PRIVATE_KEY(-----BEGIN ... KEY-----), PASSWORD
- 고위험 신원(차단): RRN(주민번호), FOREIGN_REG(외국인등록번호 7번째 5~8), PASSPORT(M/S+8자리),
  DRIVER_LICENSE(11-19-123456-01), CARD(카드번호)
- 연락/식별(마스킹): EMAIL, PHONE(010-...), KR_ACCOUNT(계좌), BIZ_NO(사업자번호)
- 문맥(마스킹): PERSON(한국이름), ADDRESS(도로명주소), ORGANIZATION(회사/은행)
- 서버정보(마스킹): IP_ADDRESS(IPv4), HOSTNAME(내부 호스트명 *.internal/.corp/.local)

[엄격 규칙 — 반드시 지킬 것]
1. 체크섬이 필요한 카테고리(RRN, CARD, BIZ_NO)는 **아래 '유효값 풀'에서만** 골라 쓴다(직접 숫자를 지어내지 말 것 — 무효 체크섬은 탐지기가 일부러 안 잡는다).
2. 라벨한 값은 텍스트 안에 **글자 그대로** 등장해야 한다(공백/하이픈까지 동일).
3. PII는 **자연스럽게** 문맥에 녹인다(이름은 "제 이름은 X입니다", "주문자 X님" 등 다양하게).
4. 각 항목에 PII/시크릿을 5~12개 섞고, 서버 로그면 IP·호스트명도 포함한다.
5. PII처럼 보이지만 PII가 아닌 값(주문번호 ORD-2024-0613, 버전 v1.2.3, 포트 8080 등)을
   1~3개 'traps'로 넣는다(오탐 측정용).
6. ORGANIZATION/공개도메인(gmail.com, api.openai.com)은 라벨하지 않는다(마스킹 대상 아님 또는 NER 변동).

[유효값 풀 — RRN/CARD/BIZ_NO는 여기서만 사용]
RRN : 710310-4151262, 120923-1591783, 910722-3023658, 940413-4961351, 700523-4376198
CARD: 4485-2538-8853-9336, 4633-8750-0474-3953, 4755-1313-7353-7994, 4075-1163-7265-1675, 4612-2202-9729-9758
BIZ : 434-67-77758, 189-24-78001, 688-17-36719, 751-47-92277, 898-08-21922
(API_KEY/AWS/JWT/PASSWORD/PHONE/EMAIL/계좌/주소/이름은 자유롭게 지어내도 됨.)

[출력 형식 — 오직 JSON 배열만, 설명/마크다운 금지]
[
  {
    "id": 1,
    "kind": "{KIND}",
    "title": "짧은 제목(예: VOC 01 · 결제오류)",
    "text": "여기에 실제 텍스트 전문(개행은 \\n).",
    "expected": [["PERSON","김민준"], ["PHONE","010-1234-5678"], ["RRN","710310-4151262"]],
    "traps": ["ORD-2024-0613", "v1.2.3"]
  }
]

지금 위 규칙대로 {N}개를 생성해 JSON 배열만 출력하라.
```

## 2. 결과를 파일로 저장

LLM이 출력한 JSON 배열을 그대로 파일로 저장한다:

```bash
# 예: 생성 결과를 붙여넣어 저장
cat > validation/external_cases.json   # ← 여기에 JSON 붙여넣고 Ctrl-D
```

여러 번(VOC 30 + LOG 30) 돌렸다면 두 JSON 배열을 하나로 합쳐도 된다(같은 파일에 한 배열로).

## 3. PII-Guard에 입력해 채점

```bash
cd /Users/ho/workspace/Monoly_genAI/pii_guard
PYTHONPATH=. .venv/bin/python validation/load_external_test.py validation/external_cases.json
```

→ 산출물:
- `validation/EXTERNAL_REPORT.md` — 재현율/정밀도 + 카테고리별 + 항목별 + **부록(전문·검출/미검출)**
- `validation/external_log.txt` — 증거 로그

채점 방식(내장 efficacy 하니스와 동일):
- **TP**(검출) / **FN**(미검출, =빠져나간 민감정보) / **FP**(오탐, traps와 비교)
- 시크릿류는 SECRET 클래스 내 호환 매칭(github→API_KEY 등 흡수)

## 4. 자주 나오는 함정 (LLM이 만든 데이터의 흔한 문제)

| 증상 | 원인 | 대응 |
| :-- | :-- | :-- |
| RRN/카드가 다 '미검출' | LLM이 무효 체크섬 숫자를 지어냄 | 프롬프트의 '유효값 풀'만 쓰게 강제(규칙 1) |
| 라벨 값이 '미검출'인데 텍스트엔 있음 | 라벨↔텍스트 글자 불일치(공백/하이픈) | 규칙 2 강조, 로더가 부분일치로 흡수하나 정확히 일치 권장 |
| ORGANIZATION 미검출 많음 | NER 모델 한계(회사명 일부) | 정상 — ORG는 라벨 최소화하거나 변동 감안 |
| 오탐(FP) 많음 | traps가 실제 PII거나 코드 토큰 | traps는 '확실히 PII 아닌' 값만 |

> 요약: **프롬프트로 "텍스트 + 정답 라벨 + 유효값 풀"을 함께 받고**, `load_external_test.py`로 PII-Guard에 먹여 채점한다.
